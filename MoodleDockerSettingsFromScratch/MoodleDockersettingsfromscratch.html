UP2University LMS docker settings from scratch

This document contains a description of the settings that a person should configure in order to setup a Moodle instance from scratch.

The repository contains templates for the environment variables. Here you should find some information about the values of these variables.

For setting Moodle => Site administration => Development =>Debugging. When set to '1' the error reporting will go to the HTML page. This is practical, but breaks XHTML, JS, cookies and HTTP headers in general.  Set to '0', it will send the output to your server logs, allowing better debugging. The PHP setting error_log controls which log this goes to.

MOODLE_DEBUG_DISPLAY=0

For setting up SMTP emailing from Moodle, Site administration, Server, Email, Outgoing mail configuration. Give the full name of one or more local SMTP servers that Moodle should use to send mail (eg 'mail.a.com' or 'mail.a.com;mail.b.com'). To specify a non-default port (i.e other than port 25), you can use the [server]:[port] syntax (eg 'mail.a.com:587').  For secure connections, port 465 is usually used with SSL, port 587 is usually used with TLS, specify security protocol below if required. If you leave this field blank, Moodle will use the PHP default method of sending mail.

MOODLE_MAIL_HOSTS=\"\"

If SMTP server requires secure connection, specify the correct protocol type. Moodle Options at time of writing: 'none', 'ssl' 'tls'

MOODLE_MAIL_SECURITY=\"tls\"

This sets the authentication type to use on smtp server. Moodle Options at time of writing: 'LOGIN', 'PLAIN', 'NTLM', 'CRAM-MD5'

MOODLE_MAIL_AUTHTYPE=\"LOGIN\"

If you have specified an SMTP server above, and the server requires authentication, then enter the username here.

MOODLE_MAIL_USER=\"\"

If you have specified an SMTP server above, and the server requires authentication, then enter the password here.

MOODLE_MAIL_PASS=\"\"

Maximum number of messages sent per SMTP session. Grouping messages may speed up the sending of emails. Values lower than 2 force creation of new SMTP session for each email.

MOODLE_MAIL_SESSION_LIMIT=1

Cernbox.env

Cernbox is the cloud-based storage, synchronisation, and sharing service developed at CERN. To make it accessible via Moodle, there are some configuration steps that are needed in both services.

Register Moodle in CERNBox

Moodle connects to CERNBox via oAuth, so a new client needs to be registered in order to obtain the secrets that will allow the connection.

Enter CERNBox with an admin account (make sure you have set the CERNBox environment variable “ADMIN_USERS” with the comma separated list of usernames), then go to “Settings” > “User authentication” and add your Moodle installation as a client:

Name

Redirection URI

Your Moodle name

Your Moodle URL + /admin/oauth2callback.php

For example, if your users reach Moodle at https://moodle.example.com, your redirection URI would be https://moodle.example.com/admin/oauth2callback.php. The name can be chosen freely, but note that it will presented to CERNBox users, so the name should be self-explanatory to them.

After adding the client, the table displays a corresponding Client Identifier and a secret. Those will be required for the configuration in Moodle, so keep them at hand.

Register CERNBox in Moodle

The information collected from the previous step now needs to be configured into Moodle.

The following example configuration assumes a CERNBox accessible via the URI cernbox.example.com (thus, CERNBox web interface accessible in cernbox.example.com/cernbox).

CERNBOX_REPOSITORY_TYPE='owncloud'

CERNBOX_REPO_NAME='CERNBox'

Authorisation settings

CERNBOX_TOKEN_ENDPOINT='https://cernbox.example.com/cernbox/index.php/apps/oauth2/api/v1/token'

CERNBOX_AUTHORIZE_ENDPOINT='https://cernbox.example.com/cernbox/index.php/apps/oauth2/authorize'

Web Distributed Authoring and Versioning endpoint settings

CERNBOX_WEBDAV_ENDPOINT='https://cernbox.example.com/cernbox/desktop/webdav'

Open collaboration services endpoint settings

CERNBOX_OCS_ENDPOINT='https://cernbox.example.com/cernbox/ocs/v1.php/apps/files_sharing/api/v1/shares'

Name and URL settings

CERNBOX_NAME=CERNBox\ Test

CERNBOX_URL='https://cernbox.example.com/cernbox'

Authorisation settings (copy the secrets generated by CERNBox in the previous step)

CERNBOX_CLIENT_ID='ClientIdFromPreviousStep'

CERNBOX_CLIENT_SECRET='SecretFromPreviousStep'

Selcont.env

SeLCont is an easy to use toolkit, which lets teachers synchronize their lectures

(audio and video) with the accompanying slides,

The suffix of the file containing sql settings ( the prefix is “template-”, the file is in the mysql config folder ).

SELCONT_MOODLE_SQL_CONFIG="selcont.sql"

Security settings.

These setting are related with the LTI integration with the Moodle. These values can be obtained from the WordPress Multisite instance that is needed for SeLCont. LTI integration is based on the  Key and Password values are created by the admin in the WordPress instance. Regarding the service salt key, this can be found in the Wordpress Multisite wp-config.php file.

SELCONT_KEY=''

SELCONT_PASSWORD=''

SELCONT_SERVICE_SALT=''

Also used as database key

Connection settings

SELCONT_URL is the URL of the WordPress Multisite instance where SeLCont plugin has been installed.

SELCONT_URL=''

SELCONT_KEY_URL=http://${SELCONT_URL}/${SELCONT_KEY}
